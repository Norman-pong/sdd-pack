## 填写指南

### 问题类型

- **矛盾**:两处或多处说的不一致。必须引用两个位置(位置 A vs 位置 B)
- **缺定义**:重要概念(字段类型、策略、阈值)只提了名没定义
- **悬空引用**:指向不存在的章节、表、API、接口
- **模糊**:多个理解方向都合理,需要拍板定一种

### 自审清单(自己检查是否充分)

- [ ] PRD §3 全部 P0/P1 功能都交叉对比过了?
- [ ] PRD §4 NFR 与 §3 任务的依赖关系审查了?
- [ ] PRD §7 集成需求 与 §3 任务的外部依赖审查了?
- [ ] 所有"v1 做 / v1 不做"的决策互相打架吗?
- [ ] 所有验收标准都有可测阈值?
- [ ] 所有"参见 §X"的引用都能指向真实存在的章节吗?
- [ ] 如果问题清单是空的——是 PRD 真的完美,还是我审得不够深?考虑让子代理再审一遍。

### 关键差异:与 sdd-prd 阶段 1 的对照

| 维度 | sdd-prd 阶段 1 | sdd-phase 阶段 1 |
|---|---|---|
| 输入 | spec(用户故事/客户需求) | PRD(已写好) |
| 重点 | 范围/数据/部署 | 范围/依赖/验收/引用 |
| 影响 | 后续阶段产物 | 任务拆解 + 依赖链 |
| 产出后续 | ADR + 约束集 | 任务分解 + 里程碑 |
