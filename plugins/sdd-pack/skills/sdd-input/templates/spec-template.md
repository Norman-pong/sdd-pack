## 模板填写指南

### 每个章节的来源

| 章节           | 来自阶段        | 不可缺省的原因                                   |
| -------------- | --------------- | ------------------------------------------------ |
| 1. 问题陈述    | 阶段 1          | sdd-prd 阶段 1 自审要对照"问题 vs 方案"          |
| 2. 目标用户    | 阶段 1          | 不写用户角色就无"用户"段可对照                   |
| 3. v1 功能边界 | 阶段 2          | v1 必做/推迟/不做 三分法是 sdd-prd 阶段 3 的输入 |
| 4. 核心场景    | 阶段 1          | 用用户语言描述的流程是验收标准的源头             |
| 5. 验收标准    | 阶段 2          | sdd-prd 阶段 4 要把验收标准转成实施检查清单      |
| 6. 非功能需求  | 阶段 1 + 阶段 3 | 不写非功能，PRD 阶段 3 的"实施约束"会无依据      |
| 7. 未决假设    | 阶段 3          | sdd-prd 阶段 2 会优先质疑这些                    |
| 8. 选型未决    | 显式声明        | 防止选型过早固化                                 |
| 9. 参考材料    | 全部            | 保留可追溯性                                     |
