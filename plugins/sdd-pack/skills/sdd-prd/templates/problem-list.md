## 填写指南

### 问题类型

- **矛盾**：两处或多处说的不一致。必须引用两个位置（位置 A vs 位置 B）
- **缺定义**：重要概念（字段类型、策略、边界）只提了名没定义
- **悬空引用**：指向不存在的章节/表/接口
- **模糊**：多个理解方向都合理，需要拍板定一种

### 自审清单（自己检查是否充分）

- [ ] 功能章和 API 章交叉对比过了？
- [ ] 部署章和技术选型章交叉对比过了？
- [ ] 数据模型和业务规则交叉对比过了？
- [ ] 所有"v1 做 / v1 不做"的决策互相打架吗？
- [ ] 所有 P0/P1/P2 字段都有类型和约束定义吗？
- [ ] 所有"参见 §X"的引用都能指向真实存在的章节吗？
- [ ] 如果问题清单是空的——是 spec 真的完美，还是我审得不够深？考虑让子代理再审一遍。
