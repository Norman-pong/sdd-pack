## 深度审视的核心思维

阶段 2 的独特之处：它不是找 spec 写出了什么矛盾（那是阶段 1），而是问 spec 之外的"**有没有更好的路**"。

spec 是一种**关于"应该怎么做"的惯性**——它承载了某种路径依赖。也许 spec 作者因为历史原因选了某个框架，或"别人都这么写"所以写了某个方案。深度审视就是在不推翻 spec 意图的前提下，**挑战实现路径**。

核心问题永远只有一句：
> **"为什么必须这样做？有更轻、更独立、更适合团队和部署目标的方法吗？"**

## 决策原则先于选型

### 为什么先定原则

很多选型纠结是因为"A 优势 X，B 优势 Y"——如果比较维度很多，就会陷入无限对比。**决策原则是"过滤器"——它把比较维度减少→真正重要的 2-3 个。**

### 如何建立决策原则

从项目的实际情况中提炼——

**来源 1：部署约束**
- 私有化部署 → "必须能在无外网环境运行"
- 客户自有 IDC → "不能依赖云服务"
- 离线更新 → "必须支持自升级机制"

**来源 2：团队能力**
- 团队有 Rust 但没人熟 C++ → "栈必须在团队当前能力范围内"
- 前端 3 人 React → "非 React 方案必须有压倒性优势才考虑"

**来源 3：架构愿景**
- "任何组件可独立替换"
- "前端不绑定后端框架"
- "存储不绑定 ORM"

### 原则的写法

原则是**测试**——用它去测每个选型：
> "如果换掉这个组件，其他组件受影响吗？"

原则不要超过 5 个。多了等于没原则。

## 质疑的 4 个切入点

### 1. 质疑"全家桶"
**信号**：spec 选了某个全家桶（Modern.js、Next.js、Spring Boot 全家桶、Django）
**质疑**：全家桶里哪些是真正需要的？拆开用散件 + 约定，是否更轻？

**示例**：
spec 选 Modern.js → 实际上 Hono 做 API、React Router 做路由 → Modern.js 只是上层封装
→ 去掉 Modern.js，用 Vite + 少量约定 → 更轻、组件更独立

### 2. 质疑"看起来理所当然的"
**信号**：选型中没有解释"为什么"——仿佛这就是唯一合理答案
**质疑**：有没有反直觉的替代方案？

**示例**：
spec 说"用 MySQL 因为数据量大" → 实际预期数据 ≤ 100W 行
→ SQLite（含 WAL 模式）在百万级数据完全够用 → 部署更简单，零运维

### 3. 质疑"规模不匹配"
**信号**：用了"大规模"方案处理"小规模"问题，或反之
**质疑**：现在和 2 年后的规模分别是什么？方案什么时候会瓶颈？

**示例**：
spec 选 Elasticsearch 做全文搜索（10K 条文章）→ PG FTS 完全够
→ 去掉 ES，减少运维复杂度

### 4. 质疑"非功能性瓶颈"
**信号**：方案的运维/升级/调试路径含糊
**质疑**：如果出问题怎么定位？怎么回滚？怎么升级？

**示例**：
spec 说"微服务"但没有服务发现/追踪/日志聚合方案
→ 如果团队只有 5 人，"微服务"可能是过度工程 → modular monolith 更合适

## ADR 的写作心法

### ADR 不是作文——是决策的"证据链"

一份好的 ADR，读者只有 2 个问题：
1. "为什么选了 X？" — 看决策原因
2. "代价是什么？" — 看影响

不需要文学修辞，不需要背景铺垫。简明即可。

### 推翻选型的 ADR 必须写

**推翻一个 spec 继承的选型是正常操作**——但不记录推翻原因的 ADR，下一轮又会重新考虑被推翻的选型。

ADR 的存在意义之一就是 **"防止决策重做"**。

### ADR 的编号和生命周期

- 用递增序号：ADR-001, ADR-002, ...
- 一旦写定，不再修改内容
- 如果后来推翻 → 新建 ADR-00X，状态为"已推翻"，覆盖旧 ADR 状态
- 永远可以追溯"为什么那个时候做了那个决策"——这才是 ADR 的价值

## 团队经验和外部依赖的考量

除了纯技术对比，还应该考量：

### 团队经验
- "我们团队有人写过吗？"——如果 0 人用过 Rust，选 Rust 写核心服务是巨大风险
- "出了问题谁修？"——选小众框架意味着社区帮助少，所有 bug 都是自己的 bug

### 外部依赖风险
- "如果这个 SaaS/PaaS 挂了/涨价了/下线了，方案还跑得动吗？"
- "版本兼容性承诺多久？"——依赖库的 breaking change 频率

### 部署路径
- "私有化部署需要什么？"——是否需要 Docker、K8s、外部镜像仓库？
- "离线升级怎么搞？"——如果是私有化部署，升级包怎么传递？
