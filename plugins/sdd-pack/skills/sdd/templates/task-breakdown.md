## 任务详情

### T001: 设计 users 表 schema + migration

**任务描述**:
基于 PRD §3.2.1 用户注册功能,设计 users 表,含字段: id, email, password_hash, created_at, updated_at。

**验收标准**:

- [ ] migration 文件可执行(migration up/down)
- [ ] 字段类型、约束与 PRD §3.2.1 一致
- [ ] 无 N+1 隐患(关联表预留)

**涉及文档**:

- [PRD §3.2.1 用户注册](../prd/2026-06-23-user-authentication.md)
- [ADR-001 DB 选型](../architecture/decisions.md#adr-001)

**备注**:
email 字段需要 unique constraint(对齐业务规则)。

---

### T002: 实现邮箱唯一性索引

**任务描述**:
基于 T001 的 schema,加 unique index on users.email。

**验收标准**:

- [ ] migration 文件可执行
- [ ] unique index 在 email 字段上
- [ ] 重复 email 插入返回数据库错误

**涉及文档**:

- T001 schema

**备注**:
与 T001 合并可节省 0.5 人天(可选拆分)。

---

### T003: 实现 POST /api/v1/users 端点

**任务描述**:
基于 PRD §3.2.1,实现用户注册 API 端点。

**验收标准**:

- [ ] POST /api/v1/users 返回 201/400/409
- [ ] 密码使用 bcrypt 哈希(cost=12, 对齐 §4.2 NFR)
- [ ] 邮箱重复返回 409(对齐业务规则)
- [ ] 单元测试覆盖率 ≥ 80%
- [ ] 集成测试通过

**涉及文档**:

- [PRD §3.2.1 用户注册](../prd/2026-06-23-user-authentication.md)
- [PRD §4.2 安全要求](../prd/2026-06-23-user-authentication.md)
- [ADR-001 前后端选型](../architecture/decisions.md#adr-001)

**备注**:

- 外部依赖:邮件服务(由 T004 调用)
- API 文档自动生成(对齐 conventions.md §4 NFR 工具链)
