## ADR 实际填写示例

### ADR-001：前端构建——Vite vs Modern.js

**状态**：已采纳

**日期**：2026-06-10

- 完整决策记录 → `docs/architecture/decisions.md`
  **决策**：用 Vite + React Router + Hono API 层，不用 Modern.js 全家桶。

**背景**：

- spec 默认选了 Modern.js 作为前端框架
- 实际分析发现：Hono 已在选型中作为 API 层，React Router 作为路由，Modern.js 只是在上层套了一层——没有提供额外价值

**备选方案**：
| 方案 | 优势 | 劣势 |
|------|------|------|
| A: Vite + React Router + Hono（选定） | 组件可独立替换；Vite 换 rspack 不影响 Hono；Hono 换 Fastify 不影响前端 | 缺少 Modern.js 的一套内置约定 |
| B: Modern.js 全家桶（原方案） | 开箱即用；模块联邦 | 把构建、路由、API 绑定在一起；未来换构建工具成本大 |
| C: Next.js App Router | 成熟生态；SSR 优化 | 私有化部署太重；路由和 API 模式与 spec 需求不匹配 |

**决策原因**：

1. **组件可独立替换**：Vite 换 rspack 不影响 Hono；SQLite 换 Postgres 不影响 React Router
2. **私有化部署友好**：无外网依赖，纯静态构建 + 轻量 API 服务
3. **团队能力匹配**：团队有 Vite + React Router 经验，Modern.js 是盲区

**影响**：

- 正面：部署轻量（静态 + API），无框架锁定
- 代价：需要手动搭建工程规范（lint、format、test）——Modern.js 这部分内置
- 对下游：API 层保持 Hono，前端路由用 React Router，两者只在约定上耦合

**决策原则引用**：任何组件可独立替换；私有化部署可行

## 需要质疑的选型类别（自检清单）

对 spec 中每个选型，用以下问题过一遍：

### 框架/库选型

- [ ] 这个框架的"全家桶"特性是否把不该绑定的组件绑在一起了？
- [ ] 去掉这个框架，用更轻的东西 —— 缺了什么？能否自己补？
- [ ] 团队有人实际用过吗？还是只是 spec 写了？

### 存储选型

- [ ] 读写模式是什么？（OLTP/OLAP/混合）
- [ ] 10x 数据量增长后还撑得住吗？
- [ ] 私有化部署能跑吗？依赖外部服务（如 S3、Cloud SQL）吗？
- [ ] 备份、迁移、回滚方案考虑了吗？

### 通信选型

- [ ] REST/GraphQL/gRPC/WebSocket——为什么这个？场景匹配吗？
- [ ] 断网、重连、超时处理有方案吗？
- [ ] 版本化策略有吗？（API v1 → v2 怎么过渡？）

### 部署选型

- [ ] 只有 Docker 方案？裸机部署考虑了吗？
- [ ] CI/CD 依赖外部服务吗？私有化部署时 CI/CD 怎么跑？
- [ ] 自升级机制有吗？（尤其是无外网环境）
