**日期**: 2026-06-24
**环境**: omp v16.1.16, native rules 4 个全移至 `~/.omp/agent/rules/disabled/`
**装载**: `omp --hook ./plugins/sdd-pack/hooks/index.ts`

## 验证矩阵

| # | 触发场景 | 期望 | 实际 | 结论 |
|---|---|---|---|---|
| a | session_start reminder | model 看到 lore 协议摘要 + "来自 plugin hook" 标记 | ✅ model 明确说"本回复来自 sdd-pack v1.1.0 plugin hook" | **PASS** |
| b | docs/ write tool_call | sdd-doc-edit-guard 提示可见 | ⚠️ 单元测试触发,model 视角未到达 | **部分 PASS**(功能层 OK,model 视角受限) |
| c | bash commit tool_call | docs-update-guard + lore-commit-guard 提示 | ⏸️ 未直接测(commit-review 拦截优先) | n/a |

## 已知限制

- **tool_call 提示用户可见(model 视角不可见)**: omp v16.1.16 的 `pi.sendMessage` 在 tool_call 事件后的注入目标是 UI 层(TUI 消息流),不是 model 的 system prompt。session_start 不同时机,model 可见。
- **影响**: docs-update-guard / lore-commit-guard / sdd-doc-edit-guard 在 tool_call 时只对用户终端可见,model 不会自动重新决策。
- **绕过**: 在 `~/.zshrc` 配置 `alias omp='omp --hook ...'` 后用户主动看 TUI 消息;或用 native rules 兜底(B1.7 期间已通过 disabled/ 目录关闭 native)。

## 决策

- **B1.7 通过**(核心目标: native rules 移除后 plugin hook 接管,且 session_start 提示在 model 视角完整生效)
- tool_call 提示的"用户可见 / model 不可见"是 omp hook API 限制,v1.2 再优化(可能需改用 `pi.on("tool_result")` 注入 model,或等待 omp 上游改进)
- native rules 仍保留在 `disabled/`,不删除(回退保险 + 与 ADR-006 §副作用 一致)
