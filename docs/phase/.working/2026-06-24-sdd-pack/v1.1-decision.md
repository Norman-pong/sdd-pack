## 1. B0 实测结果

| 模式                    | 命令                                                     | 物理产物                                                                      | hook 触发                                       |
| ----------------------- | -------------------------------------------------------- | ----------------------------------------------------------------------------- | ----------------------------------------------- |
| marketplace install     | `omp plugin install probe@hook-probe`                    | `~/.omp/plugins/cache/plugins/hook-probe___probe___0.0.1/hooks/index.ts` 存在 | ❌ `[probe] hook loaded` **不出现**             |
| local link              | `omp plugin link /tmp/hook-link/plugins/probe`           | `~/.omp/plugins/node_modules/probe -> /tmp/hook-link/plugins/probe` symlink   | ❌ `[probe] hook loaded` **不出现**             |
| **CLI flag (直接加载)** | `omp --hook /tmp/hook-link/plugins/probe/hooks/index.ts` | n/a(直接传路径)                                                               | ✅ `[probe] hook loaded session_start` **出现** |

## 2. 决策

走 plan §Assumptions 未列出但用户确认的 **CLI flag 路径**(新选项):

- **B0 决策**: 进入 B1(不等上游修复)
- **实现调整**:
  1. `plugins/sdd-pack/package.json` **不写** `omp.hooks` 字段(omp 不识别,写了反而误导)
  2. `plugins/sdd-pack/hooks/index.ts` 保留(4 工厂聚合)作为静态资产
  3. **README 必须更新**:安装命令从 `omp plugin install sdd-pack@sdd-pack` 改为 `omp --hook ./plugins/sdd-pack/hooks/index.ts plugin install sdd-pack@sdd-pack`(让 hook 真的被加载)
  4. 或:启动脚本 alias `alias omp='omp --hook ~/.omp/plugins/cache/plugins/.../hooks/index.ts'`(持久方案)
- **PRD 验收更新**:
  - §0 业务第 3 条(rules 自动生效)— 勾选(等价实现)
  - §0 技术第 3 条(rules 被 omp-plugins provider 发现)— 改为「hook 通过 CLI flag 加载」,语义修正
