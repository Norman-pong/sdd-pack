## 1. 阶段目标

### 1.1 阶段定位

本阶段将当前散装在 `~/.agents/skills/` 与 `~/.omp/agent/rules/` 的 SDD 技能家族 + lore 提交规则整合为 omp marketplace 插件 `sdd-pack`，实现「一条命令安装、版本化管理、按需启用/禁用」。

### 1.2 阶段目标

- 完成 4 阶段（验证 → 组装 → 上线 → 完善）的全部任务，对应 PRD §9.1。
- 满足 PRD §0 目标验收开关的 12 条业务/技术验收标准。
- 解决 PRD §10.1 列出的 6 项已知风险中的高影响项（rules 发现、provider 路径）。

### 1.3 完成标准

- [ ] 阶段 1（M1）rules 发现验证通过或 fallback 方案就绪
- [ ] 阶段 2（M2）plugin 目录组装完成，含 `marketplace.json` + 4 skills + 4 rules + docs-check.sh + README.md + package.json
- [ ] 阶段 3（M3）GitHub repo `norman/sdd-pack` 创建，本地 `omp plugin install` 全流程验证通过
- [ ] 阶段 4（M4）README.md 完善，git tag `v1.0.0`
- [ ] PRD §8 验收清单 13 条全部勾选
- [ ] PRD §0 业务验收 5 条 + 技术验收 5 条全部勾选

## 5. 验收

### 5.1 验收清单

对齐 PRD §0 目标验收开关 + §8 验收标准。

#### 业务验收（PRD §0）

- [ ] 新用户通过 `omp plugin marketplace add` + `omp plugin install` 两条命令完成 SDD 全套安装
- [ ] 安装后 4 个 skill 自动出现在 omp 技能列表中
- [ ] 安装后 4 个 rule 自动生效（lore-protocol alwaysApply、其余 TTSR 拦截）
- [ ] `omp plugin disable sdd-pack@sdd-pack` 可一键禁用，不影响其他插件
- [ ] `omp plugin upgrade sdd-pack@sdd-pack` 可拉取最新版本

#### 技术验收（PRD §0）

- [ ] marketplace.json 通过 omp catalog 校验
- [ ] plugin 目录结构符合 omp 规范（skills/ + rules/ 布局）
- [ ] rules 从 marketplace 安装后被 omp-plugins provider 发现
- [ ] docs-check.sh 随插件分发并可执行
- [ ] 本地 `omp plugin link` 开发模式可用

#### 功能验收（PRD §8.1）

- [ ] `omp plugin marketplace add norman/sdd-pack` 成功拉取 catalog
- [ ] `omp plugin install sdd-pack@sdd-pack` 成功安装
- [ ] 安装后 `omp plugin list` 显示 sdd-pack，状态 enabled
- [ ] 启动 omp 后系统提示包含 4 个 sdd-* skill 的 name + description
- [ ] `skill://sdd-core` 可读取 SKILL.md 内容
- [ ] `skill://sdd-core/references/docs-check.sh` 可读取脚本内容
- [ ] `rule://lore-protocol` 可读取 rule 内容
- [ ] 执行 `git commit` 时 lore-commit-guard 规则触发
- [ ] 执行 `write docs/prd/test.md` 时 sdd-doc-edit-guard 规则触发
- [ ] `omp plugin disable sdd-pack@sdd-pack` 后 skills 和 rules 不再加载
- [ ] `omp plugin enable sdd-pack@sdd-pack` 后恢复
- [ ] `omp plugin link ./plugins/sdd-pack` 本地开发模式可用

#### 非功能验收（PRD §8.2）

- [ ] docs-check.sh 在 marketplace 安装后仍可执行（bash 3.2 兼容）
- [ ] plugin 不含 JS/TS extension modules（纯静态文件）
- [ ] 安装不需要 npm install（无 node_modules 依赖；T003 fallback 触发时除外）
- [ ] README.md 包含完整的安装/验证/开发说明
