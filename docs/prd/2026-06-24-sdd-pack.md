# SDD Pack (omp marketplace 插件) PRD

> 状态：草稿
> 修改记录：执行 `lore log docs/prd/2026-06-24-sdd-pack.md`
> 对应阶段: [SDD Pack 阶段文档](../phase/2026-06-24-sdd-pack.md)

## 0. 目标声明

将散装在 `~/.omp/agent/` 下的 SDD 技能家族(sdd-core/sdd-input/sdd-prd/sdd-phase)和相关 rules(lore-protocol/docs-update-guard/lore-commit-guard/sdd-doc-edit-guard)整合为一个 omp marketplace 插件 `sdd-pack`,实现一条命令安装、版本化管理、按需启用/禁用。

## 0. 目标验收开关

### 业务验收

- [ ] 新用户通过 `omp plugin marketplace add` + `omp plugin install` 两条命令完成 SDD 全套安装
- [ ] 安装后 4 个 skill 自动出现在 omp 技能列表中
- [ ] 安装后 4 个 rule 自动生效(lore-protocol alwaysApply、其余 TTSR 拦截)
- [ ] `omp plugin disable sdd-pack@sdd-pack` 可一键禁用,不影响其他插件
- [ ] `omp plugin upgrade sdd-pack@sdd-pack` 可拉取最新版本

### 技术验收

- [ ] marketplace.json 通过 omp catalog 校验
- [ ] plugin 目录结构符合 omp 规范(skills/ + rules/ 布局)
- [ ] rules 从 marketplace 安装后被 omp-plugins provider 发现(验证关键风险)
- [ ] docs-check.sh 随插件分发并可执行
- [ ] 本地 `omp plugin link` 开发模式可用

## 1. 背景与目标

### 1.1 业务背景

当前 SDD 系列技能和 rules 散装在两个位置:

| 组件 | 当前位置 | 问题 |
|---|---|---|
| 4 个 SKILL.md + references + templates | `~/.agents/skills/sdd-*/` | 无版本管理,分发靠手动拷贝 |
| 4 个 rules | `~/.omp/agent/rules/*.md` | 与其他 rules 混放,无法整体启用/禁用 |
| docs-check.sh | `skills/sdd-core/references/` | 随技能走,但不在 omp 标准分发路径 |

痛点:
- **分发困难**:其他用户要用 SDD 体系,需要手动拷贝 skills 和 rules 两个目录,容易遗漏
- **无版本管理**:技能更新后,已安装的用户无法通过 `upgrade` 获取新版
- **无法隔离**:SDD rules 与其他 rules 共用目录,想临时禁用 SDD 只能删文件
- **依赖不完整**:skills 引用 rules(`rule://lore-protocol` 等),但两者分开放,安装时容易只装一半

### 1.2 产品目标

| # | 目标 | 衡量方式 |
|---|---|---|
| G1 | 一键安装 | 2 条命令完成全套(skills + rules + docs-check.sh) |
| G2 | 版本化分发 | plugin version 字段 + `omp plugin upgrade` 可更新 |
| G3 | 整体启用/禁用 | `omp plugin enable/disable` 一条命令控制全套 |
| G4 | 作用域可选 | 支持 user-scope(全局)和 project-scope(单项目) |
| G5 | 开发态友好 | `omp plugin link` 支持本地开发,改完即生效 |

### 1.3 成功指标

- 新用户从"知道 sdd-pack"到"完成安装并验证 skill/rule 生效" ≤ 5 分钟
- 插件版本与 skills 内容同步率 100%(每次技能变更发一个新版本)
- 安装后 `docs-check.sh` 可直接执行,无需额外配置路径

### 3.2 详细功能描述

#### 3.2.1 marketplace.json catalog

**功能说明**:在 `.omp-plugin/marketplace.json`(omp 原生路径)定义 catalog,声明 sdd-pack 插件。

**目录结构**:
```
sdd-pack/                              # GitHub repo root
  .omp-plugin/
    marketplace.json                   # catalog(omp 原生路径,优先于 .claude-plugin/)
  plugins/
    sdd-pack/                          # plugin 目录(source 指向这里)
      skills/
        sdd-core/
          SKILL.md
          references/
            conventions.md
            templates.md
            docs-check.sh
          evals/
        sdd-input/
          SKILL.md
          references/
          templates/
        sdd-prd/
          SKILL.md
          references/
          templates/
          evals/
        sdd-phase/
          SKILL.md
          references/
          templates/
          evals/
      rules/
        lore-protocol.md               # alwaysApply: true
        docs-update-guard.md           # condition: commit
        lore-commit-guard.md           # condition: commit
        sdd-doc-edit-guard.md          # scope: write/edit docs/**
      README.md
```

**marketplace.json 内容**:
```json
{
  "name": "sdd-pack",
  "owner": { "name": "norman" },
  "metadata": {
    "description": "软件开发文档体系:spec→PRD→Phase 全流程技能 + lore/文档提交规则",
    "version": "1.0.0",
    "pluginRoot": "plugins"
  },
  "plugins": [
    {
      "name": "sdd-pack",
      "description": "SDD 技能家族 + lore 提交规则,覆盖想法→spec→PRD→Phase 全流程",
      "source": "./sdd-pack",
      "category": "development",
      "keywords": ["sdd", "prd", "phase", "lore", "documentation"],
      "version": "1.0.0"
    }
  ]
}
```

**输入/前置条件**:
- GitHub 仓库已创建
- omp marketplace 机制支持 `.omp-plugin/marketplace.json` 路径(已确认:omp 优先读此路径)

**输出/后置条件**:
- `omp plugin marketplace add norman/sdd-pack` 可成功拉取 catalog
- `omp plugin install sdd-pack@sdd-pack` 可成功安装

#### 3.2.2 skills 目录布局

**功能说明**:4 个 SDD 技能按 omp 标准布局放置,每个技能一个目录,含 SKILL.md + 同级资源。

**关键约束**(来自 `omp://skills.md`):
- 技能目录为 **一层深度**:`skills/<skill-name>/SKILL.md`
- `SKILL.md` frontmatter 必须含 `name` 和 `description`(omp-plugins provider 要求 `requireDescription: true`)
- 技能资源(references/templates/evals)放在同目录下,通过 `skill://<name>/<path>` 访问
- **不支持嵌套**:`skills/sdd/sdd-core/SKILL.md` 不会被发现

**当前 skills 目录结构验证**:
```
skills/
  sdd-core/SKILL.md       ✅ 一层深度
  sdd-input/SKILL.md      ✅
  sdd-prd/SKILL.md        ✅
  sdd-phase/SKILL.md      ✅
```

#### 3.2.3 rules 目录布局

**功能说明**:4 个 rules 按 omp 标准布局放置在 `rules/` 目录下。

**关键约束**(来自 `omp://rulebook-matching-pipeline.md`):
- omp-plugins provider(priority 90)从 plugin root 的 `rules/*.{md,mdc}` 加载
- rule name = 文件名(不含 `.md`/`.mdc`)
- frontmatter 支持:`description`、`alwaysApply`、`condition`、`scope`、`interruptMode`

**rules 清单**:

| 文件 | name | alwaysApply | condition | scope | 说明 |
|---|---|---|---|---|---|
| `lore-protocol.md` | lore-protocol | true | — | — | 全局 lore 查询 + commit schema |
| `docs-update-guard.md` | docs-update-guard | — | `(git\|lore)\s+commit` | `tool:bash` | 提交前文档同步检查 |
| `lore-commit-guard.md` | lore-commit-guard | — | `(git\|lore)\s+commit` | `tool:bash` | 提交质量门 |
| `sdd-doc-edit-guard.md` | sdd-doc-edit-guard | — | — | `tool:write(docs/**), tool:edit(docs/**)` | 写 docs/ 时路由 SDD 技能 |

#### 3.2.4 rules 发现验证(关键风险项)

**功能说明**:验证 marketplace 安装的 plugin 的 rules 是否被 omp-plugins provider 发现。

**背景**:文档中有两处描述需验证:
- `plugin-manager-installer-plumbing.md`: "installed marketplace plugin cache is surfaced as plugin roots/capabilities"
- `skills.md`: omp-plugins provider 加载 "skills/ bundled next to extension packages ... under `~/.omp/plugins/node_modules`"(只提了 npm/link 路径)

**验证方法**:
1. 在 `/tmp` 建最小 marketplace 测试结构(1 个含 rules/ 的 plugin)
2. `omp plugin marketplace add /tmp/test-marketplace`
3. `omp plugin install test-plugin@test-marketplace`
4. 启动 omp,检查 rule 是否出现在系统提示或 `rule://test-rule` 可解析
5. 如果通过:确认 marketplace 安装的 rules 可被发现
6. 如果不通过:fallback 方案——将 rules 内容改写为 extension module(hook),通过 `tool_call` 事件实现拦截

**输出/后置条件**:
- 验证结果记录在 PRD 附录
- 如果不通过,启动 fallback 方案设计

#### 3.2.5 docs-check.sh 分发

**功能说明**:文档校验脚本随 sdd-core 技能的 references/ 目录分发,安装后可通过 `skill://sdd-core/references/docs-check.sh` 访问。

**约束**:
- 脚本当前位置 `skills/sdd-core/references/docs-check.sh`,插件内路径不变
- 脚本不依赖 omp 运行时,纯 bash,可在任何项目执行
- 使用时需从 skill 目录拷贝或符号链接到目标项目的 docs/ 目录

#### 3.2.6 omp plugin link 开发模式

**功能说明**:支持 `omp plugin link` 将本地开发目录链接到 omp 插件系统,改完即生效。

**操作**:
```bash
# 在 sdd-pack repo 根目录
omp plugin link ./plugins/sdd-pack

# 修改 skills/ 或 rules/ 后,重启 omp 即生效
# 开发完成后 omp plugin uninstall sdd-pack
```

**约束**(来自 `plugin-manager-installer-plumbing.md`):
- `link` 创建符号链接到 `~/.omp/plugins/node_modules/<pkg.name>`
- 需要 plugin 目录有 `package.json`(含 `name` 字段)
- link 模式下 extension modules 可加载(marketplace install 模式不行,但本插件不需要 extension modules)

- 每次技能内容变更对应一个 plugin version 递增
- git tag 与 plugin version 保持一致

## 7. 集成需求

### 7.1 omp 插件系统集成

| 集成点 | 方式 | 说明 |
|---|---|---|
| marketplace catalog | `.omp-plugin/marketplace.json` | omp 原生路径(优先于 `.claude-plugin/`) |
| skills 发现 | omp-plugins provider (priority 90) | 从 `skills/<name>/SKILL.md` 发现 |
| rules 发现 | omp-plugins provider (priority 90) | 从 `rules/*.md` 发现(**需验证**) |
| plugin 管理 | `omp plugin install/enable/disable/upgrade` | 标准 plugin 生命周期 |
| 开发模式 | `omp plugin link` | 符号链接本地目录 |

### 7.2 与现有 rules 的共存

如果用户已有 `~/.omp/agent/rules/` 下的同名 rules:

| 情况 | 行为 | 原因 |
|---|---|---|
| 用户已有 `lore-protocol.md` | native provider (priority 100) 优先 | dedup first-wins, native > omp-plugins |
| 用户无同名 rule | plugin rule 生效 | 正常路径 |

**建议**:安装 sdd-pack 后,从 `~/.omp/agent/rules/` 移除同名 rules,避免重复加载。README.md 中说明此操作。

## 10. 风险与约束

### 10.1 已知风险

| 风险 | 影响 | 概率 | 应对措施 |
|---|---|---|---|
| marketplace 安装的 rules 不被 omp-plugins provider 发现 | 高 | 中 | 阶段 1 先验证;fallback:改写为 hook extension(需 npm install 模式) |
| omp-plugins provider 的 rules 路径不是 `rules/` 而是其他 | 高 | 低 | 文档明确说是 `rules/*.{md,mdc}`;阶段 1 验证确认 |
| skills 在 plugin 中嵌套过深不被发现 | 中 | 低 | 当前布局已是一层深度,符合规范 |
| 用户已有同名 rules 导致冲突 | 低 | 中 | native provider 优先;README 说明移除同名 rules |
| omp 版本不支持 `.omp-plugin/marketplace.json` | 中 | 低 | fallback:用 `.claude-plugin/marketplace.json`(Claude Code 兼容路径) |
| `omp plugin link` 需要 package.json 但 plugin 是纯静态 | 低 | 中 | 加一个最小 package.json(仅 `name` + `version`) |

### 10.2 约束条件

- plugin 不含 extension modules(`omp.extensions`):marketplace install 模式不加载 extension modules
- plugin 不含 MCP/LSP servers:零运行时副作用
- docs-check.sh 是纯 bash:不依赖 node/python 运行时
- 命名规范:plugin name `sdd-pack`(小写+连字符,符合 omp 命名规则)
