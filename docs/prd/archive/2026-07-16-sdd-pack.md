> [!IMPORTANT] PRD 生命周期状态机（ADR-016, 6 PrdStatus + 2 ArchiveReason + 已归档终态）
> 草稿 ↔ 待评审（可灵活切换） → 已评审 → 已规划任务 → 进行中 → **已归档**（终态）
> 已归档是唯一终态，通过 ArchiveReason（已完成/已中止）记录归档原因
> 草稿/待评审 共享"灵活修改"语义；区别：草稿是概念先行无约束，待评审是经过多轮沟通后格式规范已正式
> **硬约束**：已评审 不可回退 草稿；任意状态可直接归档但不可逆；已归档 是终态

## 0. 目标声明

将 sdd-pack v1.5/v1.6 的 5 个分散 PRD(SDD Pack / sdd CLI / sdd Extension / OpenSpec Harness / 双范式架构)整合为统一总 PRD,**清空 docs/prd/ 目录的并发维护负担**,后续 v1.7+ 演进在此单文件上做 supersedes 链或 delta merge。

## 0.1 整合范围

- v1.4 hook 接管后 skills/rules/agents 退役与 v1.5 恢复正本资产
- v1.5 双范式一体化(SDD 正本 + OpenSpec 可选 hook 默认实现)
- v1.6 hook 逻辑合并进 extension module(本仓库上一阶段落地)
- v1.7 PRD 状态机重构(6 PrdStatus + 2 ArchiveReason + 已归档终态,见 ADR-016)

## 1. 背景与目标

### 1.1 背景

sdd-pack 历经 v1.0–v1.6 共 7 个版本,docs/prd/ 同时存在 5 个活跃 PRD:
- `2026-06-24-sdd-pack.md`(v1.2.3 已发布,v1.3+ 已冻结)
- `2026-06-29-sdd-cli.md`(v1.2.3 已发布)
- `2026-06-30-sdd-extension.md`(已发布)
- `2026-07-01-openspec-harness.md`(v1.0 已发布)
- `2026-07-01-sdd-dual-paradigm.md`(v1.0 已发布)

每个 PRD 维护独立的"对应阶段",`validateDocs` 报告 phase 缺失或 status 字段漂移。**多 PRD 并发维护已成负担**,尤其在 harness 化、状态机重构这类横切改动时。

### 1.2 目标

- **整合**:5 个历史 PRD 全部归档(已在 archive/),由 `2026-07-16-sdd-pack.md` 统一总览
- **单点演进**:v1.7+ 新需求在 `2026-07-16-sdd-pack.md` 上做 supersedes 或 delta,不再开新文件
- **状态机现代化**:从 7 状态(草稿/评审中/已评审/已发布/已替换/已归档/已废弃)升级到 6 PrdStatus(草稿/待评审/已评审/已规划任务/进行中/已归档) + 2 ArchiveReason(已完成/已中止),已归档作为唯一终态

### 1.3 非目标

- 不重写 hook/extension 实现(已在 v1.6 完成)
- 不引入新的 CI 流水线(已稳定)
- 不回填历史 PRD 的状态机修正(已归档,不动)
- 不在本 PRD 周期内回填 skills/rules/agents 中的旧状态机文档（已记录为已知债务）
